\subsubsection*{template$<$typename Derived$>$\newline
class Eigen\+::\+Dense\+Base$<$ Derived $>$}
\mbox{\hyperlink{classBase}{Base}} class for all dense matrices, vectors, and arrays. 

This class is the base that is inherited by all dense objects (matrix, vector, arrays, and related expression types). The common \mbox{\hyperlink{namespaceEigen}{Eigen}} API for dense objects is contained in this class.

EIGEN\+\_\+\+DEVICE\+\_\+\+FUNC void \mbox{\hyperlink{classEigen_1_1DenseBase}{Eigen\+::\+Dense\+Base}}$<$ Derived $>$\+::transpose\+In\+Place\hspace{0.3cm}{\ttfamily [inline]}}

This is the \char`\"{}in place\char`\"{} version of \mbox{\hyperlink{classEigen_1_1DenseBase_af891b9782f712151a471b3d7cfaaa2e8}{transpose()}}\+: it replaces {\ttfamily $\ast$this} by its own transpose. Thus, doing 
\begin{DoxyCode}{0}
\DoxyCodeLine{m.transposeInPlace();}

\end{DoxyCode}
 has the same effect on m as doing 
\begin{DoxyCode}{0}
\DoxyCodeLine{m = m.transpose().eval();}

\end{DoxyCode}
 and is faster and also safer because in the latter line of code, forgetting the \mbox{\hyperlink{classEigen_1_1DenseBase_a5df64c66228ba75bbc66db2584185527}{eval()}} results in a bug caused by aliasing.

Notice however that this method is only useful if you want to replace a matrix by its own transpose. If you just need the transpose of a matrix, use \mbox{\hyperlink{classEigen_1_1DenseBase_af891b9782f712151a471b3d7cfaaa2e8}{transpose()}}.

\begin{DoxyNote}{Note}
if the matrix is not square, then {\ttfamily $\ast$this} must be a resizable matrix. This excludes (non-\/square) fixed-\/size matrices, block-\/expressions and maps.
\end{DoxyNote}
