\subsubsection*{template$<$typename Expression\+Type, int Direction$>$\newline
class Eigen\+::\+Vectorwise\+Op$<$ Expression\+Type, Direction $>$}
Pseudo expression providing broadcasting and partial reduction operations. 


\begin{DoxyTemplParams}{Template Parameters}
{\em Expression\+Type} & the type of the object on which to do partial reductions \\
\hline
{\em Direction} & indicates whether to operate on columns (\mbox{\hyperlink{group__enums_ggad49a7b3738e273eb00932271b36127f7ae2efac6e74ecab5e3b0b1561c5ddf83e}{Vertical}}) or rows (\mbox{\hyperlink{group__enums_ggad49a7b3738e273eb00932271b36127f7a961c62410157b64033839488f4d7f7e4}{Horizontal}})\\
\hline
\end{DoxyTemplParams}
This class represents a pseudo expression with broadcasting and partial reduction features. It is the return type of \mbox{\hyperlink{classEigen_1_1DenseBase_abfbb90acef01249a37f7d1323a414896}{Dense\+Base\+::colwise()}} and \mbox{\hyperlink{classEigen_1_1DenseBase_ac1fbfd5b45f2b4b9bff9d1994e335863}{Dense\+Base\+::rowwise()}} and most of the time this is the only way it is explicitly used.

To understand the logic of rowwise/colwise expression, let\textquotesingle{}s consider a generic case {\ttfamily A.\+colwise().foo()} where {\ttfamily foo} is any method of {\ttfamily \mbox{\hyperlink{classEigen_1_1VectorwiseOp}{Vectorwise\+Op}}}. This expression is equivalent to applying {\ttfamily foo()} to each column of {\ttfamily A} and then re-\/assemble the outputs in a matrix expression\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{[A.col(0).foo(), A.col(1).foo(), ..., A.col(A.cols()-\/1).foo()] }

\begin{DoxyItemize}
\item If the number of columns is zero, then a 1x0 row-\/major vector expression is returned.
